# Streaming Parameter Memory (SPM) 方法描述

---

## 一、學術角度：詳細技術描述

### 研究背景與動機

大型語言模型（LLMs）在部署後面臨一個核心挑戰：**如何在持續的對話流中快速學習新知識，同時不遺忘過去已學到的資訊？** 這被稱為「穩定性-可塑性困境」（Stability-Plasticity Dilemma），也是在線學習（Online Learning）和持續學習（Continual Learning）領域的經典難題。

SPM（Streaming Parameter Memory）從認知科學中的**互補學習系統理論**（Complementary Learning Systems, CLS; McClelland et al., 1995）汲取靈感。該理論指出，人腦透過兩個互補系統實現記憶：**海馬體**負責快速編碼新經驗，**新皮質**負責緩慢整合長期知識。SPM 在參數空間中複刻了這一機制。

### 核心架構：雙流 LoRA

SPM 基於一個**凍結的基座模型**（Qwen/Qwen3.5-9B, 90 億參數），在其上疊加兩個獨立的 LoRA（Low-Rank Adaptation）適配器：

| 組件 | Working-Memory LoRA (WM) | Long-Term LoRA (LT) |
|------|--------------------------|----------------------|
| **類比** | 海馬體（快速學習） | 新皮質（穩定儲存） |
| **秩（Rank）** | 16 | 64 |
| **Alpha** | 32 | 128 |
| **目標模組** | 注意力層（q, k, v, o_proj） | 注意力層 + MLP（共 7 個投影） |
| **參數量** | ~13M | ~52M |
| **學習率** | 5×10⁻⁴ | 1×10⁻⁴ |
| **生命週期** | 每個對話會話（session）開始時歸零 | 跨會話持久保存 |
| **更新時機** | 每輪對話（per-turn）在線更新 | 僅在會話結束時的整合階段更新 |

前向推理時，模型輸出為三者之和：

$$\hat{y} = f_{\text{base}}(x) + f_{\text{LT}}(x) + f_{\text{WM}}(x)$$

WM-LoRA 採用**殘差零初始化**設計——每個新會話開始時所有 WM 參數歸零，使模型從 LT 的知識狀態出發，WM 僅捕捉會話內的增量偏差。

### 在線學習流程（Per-Turn Update）

每輪對話到來時，SPM 僅更新 WM-LoRA 的參數：

1. 將當前對話輪次格式化為 ChatML 格式（含 persona 系統提示）
2. 以**下一個 token 預測（NTP）**為目標，對 WM-LoRA 執行 3 步梯度下降（AdamW, 梯度裁剪 max_norm=1.0）
3. 將該輪數據存入**會話緩衝區（Session Buffer）**

LT-LoRA 在此過程中**不接收任何梯度**。

### 核心貢獻：行為蒸餾整合（Behavioral Distillation Consolidation）

會話結束時，SPM 執行整合操作，將 WM 中捕獲的行為知識遷移到 LT 中。整合損失函數為：

$$\mathcal{L} = \underbrace{\mathcal{L}_{\text{NTP}}(\theta_{LT}'; \mathcal{B}_{\text{reservoir}})}_{\text{保留舊知識}} + \underbrace{\beta \cdot \mathbb{E}_{x \in \mathcal{B}_{\text{session}}} \left[ D_{\text{KL}} \left( p_{\text{LT+WM}}(\cdot|x) \,\|\, p_{\text{LT'}}(\cdot|x) \right) \right]}_{\text{行為蒸餾遷移}} + \underbrace{\gamma \cdot \Omega_{\text{Fisher}}}_{\text{Fisher 信賴域}}$$

三項損失的含義：

1. **NTP 保留損失（Reservoir Replay）**：從蓄水池緩衝區（Reservoir Buffer, 容量 5000）採樣歷史數據，對 LT 計算標準語言模型損失，防止遺忘過去會話的知識。

2. **KL 行為遷移損失（核心創新）**：在整合開始前，以 LT+WM 聯合模型為「教師」，緩存其在當前會話數據上的 logits。然後切換至僅 LT 模式（「學生」），最小化師生之間的 KL 散度。關鍵洞見：**不是在參數空間合併 WM→LT，而是在行為空間蒸餾**，這避免了直接參數合併帶來的干擾。β 默認為 1.0。

3. **Fisher 信賴域（可選正則項）**：以對角 Fisher 資訊矩陣加權的二次懲罰，限制 LT 參數偏離整合前的值。γ 默認為 0.0（不啟用），作為可選穩定器。

$$\Omega_{\text{Fisher}} = \sum_{i} F_i \cdot (\theta_i^{\text{LT'}} - \theta_i^{\text{LT}})^2$$

### 記憶管理：蓄水池抽樣

歷史數據透過經典的**蓄水池抽樣**（Vitter's Algorithm R）維護在固定大小的緩衝區中。每個新樣本以 1/n 的概率替換緩衝區中的已有樣本，保證所有歷史樣本被等概率保留。會話結束後，會話緩衝區的數據合併入蓄水池。

### PPO 策略優化（輔助模組）

SPM 還訓練了一個 MLP 策略網路（~0.5M 參數），用 PPO 學習**何時觸發整合**（而非固定在每個會話邊界整合）。策略網路接收 8 維狀態向量（Fisher 量級、緩衝區充滿度、會話進度、近期損失趨勢等），輸出二值決策（等待 / 立即整合）。獎勵函數平衡三個目標：

$$r = \underbrace{\text{quality}}_{\text{損失改善} \times 2.0} - \underbrace{\text{forgetting}}_{\text{保留度下降} \times 5.0} - \underbrace{\text{cost}}_{\text{整合開銷} = 0.1}$$

> **注意**：PPO 模組目前為實驗性質，主實驗流程仍使用固定的會話邊界整合策略。

### 評測方案

在 PersonaChat 和 LIGHT 兩個流式對話數據集上，與 6 個基線方法對比：

- **Frozen**（無適應，下界）
- **Single-LoRA**（在線 NTP，無防遺忘）
- **Single-LoRA + EWC + Replay**
- **Param-Matched Single-LoRA**（r=80, ~65M, 參數量匹配）
- **Retrieval-Augmented**（無參數更新，top-k 歷史檢索）
- **Dual-LoRA + EWC**（SPM 架構但用參數空間 EWC 替代行為蒸餾）

核心指標：
- **語義保留 F1（Semantic Retention F1）**：ROUGE-L 衡量跨會話知識保持
- **適應速度（Adaptation Speed）**：會話內損失下降的相對幅度
- **遺忘率（Forgetting Rate）**：跨會話保留度的衰減速率
- **困惑度（Perplexity）**：語言建模品質
- **BLEU-4**：生成回應與參考的重疊度

## 二、通俗版：給外行人的解釋

### 一句話版本

> **SPM 讓 AI 像人一樣，一邊聊天一邊記住你說過的話，而且不會因為認識了新朋友就忘記老朋友。**

### 詳細版本

想像你有一個非常聰明的助手。這個助手讀過無數的書（這就是「大型語言模型」的預訓練），什麼都懂一點，但它有一個致命缺點：**沒有記憶**。每次你和它聊天，它都像第一次見到你一樣。你上次告訴它你喜歡貓、住在台北、是個工程師——它全忘了。

#### 問題在哪裡？

現有的解決辦法各有缺陷：
- **硬塞上下文**：把過去的對話全部塞進提示詞裡——但對話太多就塞不下了，而且很慢很貴。
- **直接微調**：讓 AI 學習新對話——但學了新的就忘了舊的，就像一塊只能寫這麼多字的白板，寫新的就得擦掉舊的。

#### SPM 怎麼解決？

SPM 模仿了**人腦的記憶方式**。你可能知道，人有兩種記憶系統：

- 🧠 **短期記憶（海馬體）**：你剛聽到的電話號碼、剛認識的人的名字——記得快，忘得也快。
- 🧠 **長期記憶（大腦皮質）**：你的母語、騎自行車的技能——學得慢，但一旦記住就很難忘。

SPM 在 AI 裡建了兩個類似的「記憶模組」：

| | 工作記憶（WM） | 長期記憶（LT） |
|---|---|---|
| **像什麼** | 便利貼 | 筆記本 |
| **做什麼** | 在這次聊天中快速記住你說的話 | 把重要的東西永久保存下來 |
| **特點** | 每次新對話都換一張新的便利貼 | 不斷積累，越來越厚 |
| **大小** | 小而輕巧（~1300 萬參數） | 大而穩固（~5200 萬參數） |

#### 關鍵：「睡眠整合」

最巧妙的部分是：**每次聊天結束後，AI 會做一次「睡眠整合」**（就像人類睡覺時大腦會整理白天的記憶一樣）。

但它不是簡單地把便利貼上的字抄到筆記本裡（那樣容易抄亂），而是做了一件更聰明的事：

1. 📝 先讓「便利貼 + 筆記本」一起回答問題，記下答案
2. 📓 然後只用筆記本去學習，讓筆記本的回答越來越接近那個「合作答案」
3. 📚 同時複習一些之前的舊筆記，確保不會忘記老朋友

這就是**行為蒸餾**——不是複製參數，而是複製「行為」。就像你不是把老師的筆記影印一份，而是聽老師講課後用自己的話重新理解一遍，記得更牢。

#### 效果

這樣一來，AI 就能：
- ✅ 和你聊天時**快速記住**你說的每一件事（工作記憶在學）
- ✅ 下次再見面時**仍然記得**你之前說過的話（長期記憶已整合）
- ✅ 同時和很多人聊天，**不會把你和別人搞混**（每個人有獨立的工作記憶）
- ✅ **不需要把歷史對話全部重新輸入**（知識已經「長進」了模型的參數裡）

#### 一個生活化的比喻

> 想像一個酒保。每天晚上有不同的客人來聊天，酒保用便利貼記下每個客人的喜好（工作記憶）。打烊後，酒保花幾分鐘把重要的東西整理到筆記本裡（長期記憶整合）。第二天便利貼扔掉，但筆記本還在。久而久之，這個酒保「認識」了所有的常客，記得每個人愛喝什麼——而且從來不會因為記了新客人的喜好就忘了老客人的。
